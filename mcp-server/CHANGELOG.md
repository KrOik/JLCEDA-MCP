## [1.1.0] - 2026-03-18

### 新增

- 侧边栏底部新增打开 EDA 时自动收起侧边栏的开关，可在 VS Code 设置中配置。

### 变更

- 优化：AI 在原理图中放置器件时，会自动将器件布局在图纸中心区域，不再出现摆到边角或超出红框的情况。
- 优化：AI 放置完器件后会自动完成所有引脚的电气连接，不再留悬空引脚要用户手动补线。
- 新增：用户说检查原理图时，AI 默认进行功能性审查（分析电路用途、器件选型、引脚连接、能否正常工作），只有明确说跑 ERC 或做 DRC 时才执行规则检查。

## [1.0.0] - 2026-03-16

### 当前功能

**多 AI 客户端支持**

- 支持在 VS Code（GitHub Copilot）中作为 MCP 服务使用。
- 支持在 Cursor 中作为 MCP 服务使用。
- 宿主类型自动识别，无需手动切换。

**侧边栏面板**

- 在 VS Code 活动栏注册「嘉立创 EDA MCP」侧边栏，实时显示服务运行状态和桥接连接状态。
- 显示已连接的 EDA 实例列表，区分活动与待命角色。

**三个 MCP 工具**

- jlceda_api_search：供 AI 检索嘉立创 EDA API 文档。
- jlceda_api_invoke：供 AI 调用嘉立创 EDA API，执行实际操作。
- jlceda_context_get：供 AI 获取当前 EDA 运行时状态（当前工程、文档、图页、选中元件等）。

**服务配置**

- 桥接监听地址和端口通过 VS Code 设置持久化，配置变更后自动重新对接 AI 客户端。

**AI 助手指令**

- 内置嘉立创 EDA 操作系统指令，AI 连接时自动获取。
- 支持在 VS Code 设置中自定义补充指令，保存后重连即生效。
